01 — The Problem

Migration at Scale Is Broken

10K+
DoD web properties requiring migration

Large institutions manage web portfolios of staggering complexity. Hundreds or thousands of properties, built on different platforms over decades, with inconsistent structures and no clear migration path.

The bottleneck has never been the destination CMS. It’s the inability to efficiently normalize and classify what already exists.

  • Migration projects scoped in years — many never complete
  • Teams spend majority of time on repetitive, low-value normalization
  • Mid-project exceptions cause scope creep and budget overruns
  • Stakeholders lose trust and begin to obstruct the process
  • 50+ FTE for 5+ years — what’s actually been proposed at scale
02 — The Solution

Five-Stage Pipeline

From raw crawl to CMS-ready manifest — deterministic rules handle the obvious, AI handles the rest.
01
Source Ingestion
Accepts URLs, sitemaps, or API endpoints. Crawls and stores raw HTML, structured data, and metadata. Media captured by reference.
02
Deterministic Normalization
Rules-based processing handles high-confidence mappings first. Strips chrome, extracts known metadata, classifies structured content.
03
AI Classification
Where rules can’t resolve ambiguity, AI identifies content types, infers relationships, and fills classification gaps in batches.
04
Confidence Scoring
Every item scored for mapping accuracy. Low-confidence items flagged for human review. Aggregate score serves as complexity indicator.
05
Manifest Output
Platform-agnostic JSON or Markdown manifest — clean, consistent, ready for any CMS. Stages to review environment before publication.

06 — Origin

Born from NASA-Scale Migration

Project Andy emerged directly from Lone Rock Point’s engagement on large-scale web migration with federal agencies — including NASA properties with tens of thousands of content items across deeply inconsistent structures.

The standard approach — manual analysis, custom extraction scripts, phased rollout — hit fundamental scalability limits. Scripts handled known patterns but couldn’t adapt when content surprised them. On a site like NASA’s, surprising content was the norm.

“The team would identify pages that appeared similar, build a script, then discover the group contained variation requiring a different approach. Constant re-scoping. Extended timelines. Stakeholder frustration.”

08 — Architecture

Two-Layer Intelligence

Andy combines deterministic rule-based processing with AI-assisted classification. Rules handle what’s confident. AI handles what’s ambiguous. Humans handle what matters.

Designed for managed cloud hosting in commercial engagements and client-controlled infrastructure for classified environments. No hard dependency on any specific hosting platform.

01
Ingestion Layer
Multi-source crawling — URLs, sitemaps, APIs. Raw storage of HTML, metadata, and media references.
02
Rules Engine
Pattern matching, metadata extraction, chrome stripping. Handles high-confidence content quickly and cheaply.
03
AI Classification
Batch processing for ambiguous content — type identification, relationship inference, gap filling.
04
Confidence Engine
Per-item scoring with threshold-based flagging. Aggregate metrics for upfront complexity assessment.
05
Manifest Generator
Platform-agnostic JSON/Markdown output. Staging environment integration for governance review.
